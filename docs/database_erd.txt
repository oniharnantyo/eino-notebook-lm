╔══════════════════════════════════════════════════════════════════════════════╗
║                           COLUMN DESCRIPTIONS                               ║
╚══════════════════════════════════════════════════════════════════════════════╝

notebooks:
  • id              : Unique identifier for the notebook
  • user_id         : Owner of the notebook
  • title           : Notebook title
  • description     : Optional description
  • content         : Optional content
  • status          : 'active', 'archived', 'deleted'
  • tags            : Array of tags for categorization
  • metadata        : Additional metadata as JSONB
  • created_at      : Timestamp when notebook was created
  • updated_at      : Timestamp when notebook was last updated
  • deleted_at      : Soft delete timestamp

conversations:
  • id              : Unique identifier for the conversation session
  • notebook_id     : Which notebook this conversation belongs to
  • metadata        : Additional session-level metadata
  • created_at      : When this conversation session was created

messages:
  • id                  : Unique identifier for the message
  • conversation_id     : Which conversation this message belongs to
  • sequence_num        : Order within conversation (1, 2, 3, ...)
  • response_id         : Identifies which turn this message belongs to
  • previous_response_id : Links to previous turn for threading
  • message             : Full message as JSONB (StoredMessage structure)
                         • Contains: role, content, extra, timestamp
                         • Content can be string or structured object
  • model               : AI model used for this turn
  • finish_reason       : Why the response ended (stop, length, etc)
  • prompt_tokens       : Input token count for this turn
  • completion_tokens   : Output token count for this turn
  • total_tokens        : Total token count for this turn
  • created_at          : When this message was created

    ↑ FOR FETCHING MESSAGES BY TURN

╔══════════════════════════════════════════════════════════════════════════════╗
║                      CASCADE BEHAVIOR                                        ║
╚══════════════════════════════════════════════════════════════════════════════╝

DELETE FROM notebooks WHERE id = 'xxx'
  → Deletes all conversations where notebook_id = 'xxx'
  → Deletes all messages where conversation_id IN (deleted conversation ids)

DELETE FROM conversations WHERE id = 'yyy'
  → Deletes all messages where conversation_id = 'yyy'

DELETE FROM messages WHERE id = 'zzz'
  → Deletes only that message


╔══════════════════════════════════════════════════════════════════════════════╗
║                        API ENDPOINTS                                       ║
╚══════════════════════════════════════════════════════════════════════════════╝

POST /api/v1/responses
  → Create/continue conversation

GET /api/v1/notebooks/{notebookId}/conversations/messages
  ?responseId={optional}   → Specific or latest conversation
  &limit=10                → Messages per page (default: 10)
  &beforeSequence={optional} → Cursor for next page (older messages)

  Returns latest 10 messages (if no beforeSequence)
  Returns next 10 older messages (if beforeSequence provided)
